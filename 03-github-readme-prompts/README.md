# GitHub README Prompts

## Module Summary
This folder focuses on README generation and documentation prompts for repositories and subfolders. Use it when you need stronger developer-facing documentation with better structure and clarity.

## Directory Structure
```text
├── folder-readme  # resource
│   └── github-folder-readme-maker.md  # prompt file
└── repo-main-readme  # resource
    └── github-main-readme-maker.md  # prompt file
```

## How To Use This Folder
- Choose `repo-main-readme` for a whole project.
- Choose `folder-readme` for one focused module or subfolder.

## Workflow Pattern
1. Pick the closest prompt instead of starting from a blank page.
2. Copy the prompt block from the chosen file.
3. Supply the source material, constraints, and target outcome.
4. Save improved variants as new versions when behavior changes materially.

## Maintenance Notes
- Keep names short, searchable, and folder-specific.
- Add new versions only when the output behavior changes clearly.
- Prefer updating this README when new files are added.
